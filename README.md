# Midi Locale Key Fix

Маленький модуль для Foundry VTT v13 + dnd5e 5.x, который чинит проблему совместимости русских переводов (`laaru-dnd5-hw`, `ru-ru`, `rus`) с автоматизациями Midi-QoL и Cauldron of Plentiful Resources.

## Проблема

При использовании русской локализации некоторые модули (в частности CPR при касте Hex/Сглаз) создают Active Effect с ключом вида:

```
flags.midi-qol.disadvantage.check.сил
```

вместо корректного:

```
flags.midi-qol.disadvantage.check.str
```

Midi-QoL такие ключи не парсит, эффект тихо игнорируется.

## Что делает модуль

При создании или обновлении любого Active Effect модуль смотрит на последний сегмент ключа в `changes`. Если это русская аббревиатура из таблицы (`сил`, `лов`, `акр`, `прн` и т.д.) — заменяет на английскую (`str`, `dex`, `acr`, `ins`).

Работает на хуках:
- `preCreateActiveEffect` — для эффектов, накладываемых заклинаниями
- `preUpdateActiveEffect` — для правок существующих эффектов
- `preCreateItem` — для предметов с встроенными эффектами

## Установка

1. Скопируй папку `midi-locale-fix` в `Data/modules/` твоей инсталляции Foundry
2. Перезапусти мир
3. В **Настройках мира → Управление модулями** включи «Midi Locale Key Fix»

## Настройки

В клиентских настройках есть переключатель **Логирование исправлений** — включает вывод в консоль (F12) каждого исправленного ключа. Полезно для диагностики.

## Расширение списка

Если в твоём переводе используются другие аббревиатуры (например, `мудр` вместо `мдр`) — открой `scripts/main.mjs` и добавь нужные пары в `ABILITY_MAP` или `SKILL_MAP`.
